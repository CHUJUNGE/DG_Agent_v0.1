# Prompt Iteration Notes - Case_001 Gap Review

## 1. 来源

- 模型输出与人工修改意见：`model_tests/case_001_gpt-5-mini_review.md`
- 正式版本：`Case_001/【Final Digital Diary DG】Gum Occasion Study - Synocodes - 250108.pdf`

## 2. 主要 Gap

### Gap 1：输出像“方案建议”，不像最终 DG 题目

模型输出里大量使用：

- 模块目的
- 对应研究问题
- 建议题量
- 建议题目示例
- 研究目的
- 设计说明

但正式 DG 更像可直接放进日记本的题目设计稿，每个模块格式清楚：

```text
模块x：xxx
引导语：
题目：
1.
2.
...
结束语：
```

需要迭代：

- 详细题目设计部分必须输出完整题目，而不是“建议题目示例”。
- 面向受访者的题目正文要优先，不要在每题后面堆研究目的/设计说明。
- 内部说明如果保留，只能简短放在模块开头或结尾，不能淹没题目。

### Gap 2：题目过度产品/品牌导向，尤其是“关于我”

模型在“关于我”里过早加入：

- 口腔/提神类产品
- 状态不在线时段
- 工作/学习与口香糖机会的连接

正式版本中，“关于我”不需要和产品/品牌发生直接关联，而是更多描摹受访者本人：

- 自我介绍
- 学业/工作
- 居住空间
- 兴趣爱好
- 社交生活
- 理想生活
- 焦虑/压力
- 生活评分

需要迭代：

- 基础画像模块先理解“人”，不要急于进入品类或产品。
- 通过人的生活方式、兴趣、空间、关系和痛点，间接帮助研究员理解潜在场景。

### Gap 3：题目太死板，缺少开放性

模型在“典型一天”和日记模块中给了太多固定框架、选项和具体例子，容易限制受访者回答。

正式版本更常用：

- 请描画一下
- 请回忆一下
- 你会怎么形容
- 哪些时刻
- 为什么
- 有什么不同

需要迭代：

- 示例只能作为轻柔提示，不应成为硬性选项。
- 不要用过度细碎的模板让受访者像填表。
- 问题要打开生活细节，而不是把答案限制在研究员预设里。

### Gap 4：模块 3 应是“正餐以外吃吃喝喝”图谱，而不是窄化为“嘴巴相关解决方案”

正式版本定义清楚：

```text
正餐以外，不以单纯“吃饱”为主要目的的食品与饮品
```

并用分类 guide 引导：

- 日常囤的
- 随手买的
- 被分享/推荐的
- 餐后时刻
- 学习/工作时刻
- 在路上时
- 临上场准备时
- 高消耗/高重复时

需要迭代：

- 模块 3 先下载更广义的正餐外吃喝图谱，再追问口香糖是否出现。
- 不要一开始就把模块命名和内容窄化为“口香糖替代品矩阵”。

### Gap 5：日记模块过重、过像事件调查表

模型把每个事件拆成 6-8 个小问，还强调每次 3-6 分钟，显得执行负担重。

正式版本更像每日自然记录，重点是：

- Doublemint functional moments：嘴巴不舒服、餐后、清新需求。
- Extra emotional moments：状态不佳/off-center、应对方式、食品饮品是否进入、口香糖是否起作用。
- 所有问题受访者都答，但 internal label 不暴露品牌。

需要迭代：

- 日记题要完整，但不要用过重的“事件模板”语言。
- 品牌分支可以作为内部结构，但受访者题面不应强调品牌标签。
- 每日记录应更自然，像陪伴式 diary。

### Gap 6：购物任务偏离正式版本重点

模型偏向“货架/收银台拍照/搜索截图/小包装模拟”，过于研究员假设化。

正式版本重点：

- 第 4-5 天布置购物任务。
- 尽量视频 + 语音记录真实购买过程，像生活 vlog。
- 高频用户自然购买可视作完成。
- 低频/流失用户不强制渠道，在最自然真实情况下完成。
- 记录购买前、购买中、购买后、卡点回顾。
- 第 5-6 天实际吃 1 次益达、1 次绿箭，记录 eating experience。

需要迭代：

- 购物任务优先还原自然购买 journey，不要强行货架审计。
- shopping task 应包含 eating experience。
- 任务说明要区分高频用户与低频/流失用户。

### Gap 7：习惯养成模块应该围绕“嘴巴小物/正餐外吃喝”里的具体产品

正式版本让受访者从自己正餐外、不以饱腹为目的的吃喝中选择至少 2 种产品：

1. 近 1-2 年新养成习惯的食品/饮品。
2. 一直以来个人最喜欢的食品/饮品。

然后追问：

- 初尝
- 习惯养成
- 让你戒不掉的点
- 中断
- 变化
- 角色

需要迭代：

- 习惯养成不要只问口香糖。
- 要从前面图谱里抽取具体产品，理解跨品类习惯机制。

### Gap 8：品牌与产品模块应放最后，且包含品牌人格化、品牌印象变化、理想产品和 stimulus

正式版本最后才进入：

- 偏好/好感品牌
- “嚼出自在好状态”的画面和关键词
- 提到益达/绿箭想到什么
- 品牌印象变化
- 把品牌比作一个人
- 理想口香糖
- stimulus 维度选择和组合

需要迭代：

- 品牌/产品期待必须放后段。
- 用 metaphor 具体化品牌。
- 产品创新题要兼顾清新口气和状态调整两个方向。

## 3. Prompt 迭代方向

### 3.1 输出格式强约束

详细题目设计必须采用：

```text
模块x：模块名
引导语：
题目：
1.
2.
...
结束语：
```

禁止用：

- 建议题目示例
- 示例题
- 建议题量
- 每题大量研究目的/设计说明

### 3.2 模块内容强约束

如果是 Case_001 或相似口香糖/嘴巴相关品类增长项目，应优先遵循正式 DG 的模块逻辑：

1. 关于我
2. 我典型的一天
3. 我正餐以外的“吃吃喝喝”
4. 我的 6 天日记
5. 购物任务
6. 我的“嘴巴习惯”养成
7. 我期待的品牌与产品

### 3.3 题目风格约束

- 受访者题面要自然、开放、有陪伴感。
- 不要给太多例子限制答案。
- 示例只在帮助理解时出现，并用“比如/如”等轻提示。
- 不要过早暴露品牌，品牌进入最后模块或任务模块。
- 对低线城市/蓝领/填写负担较高人群，优先鼓励语音、图片、视频。
