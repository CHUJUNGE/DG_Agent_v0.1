# Prompt Iteration - Case_001 v2 Review

## 本轮依据

- 用户对 `case_001_gpt-5-mini_review_v2.md` 的修改意见。
- 正式版 Final Digital Diary DG 的模块结构和题面风格。

## v2 主要 gap

1. 研究逻辑方向基本接近，但题面仍然太像研究员 checklist。
2. 每道题都加括号解释，形成“为了引导而引导”的感觉。
3. 部分题目命令感太强，例如“每段用一行写清时间和活动”。
4. 部分题目为了具体而具体，例如把典型一天直接收窄到“精神下滑/注意力不集中”。
5. 日记模块过重，包含标题、打分、强制上传等要求，不像正式 DG 的自然生活记录。
6. 购物任务偏货架审计，正式版更强调自然购买 journey 和 vlog 式记录。

## 正式版 DG 对应启发

- “关于我”先了解受访者本人，不直接进入品类、品牌或状态调节。
- “我典型的一天”用开放描画，不写成机械时间表。
- “正餐以外的吃吃喝喝”用场景提示帮助回忆，但不把每个提示拆成封闭小题。
- “6 天日记”应像真实生活记录，素材用于降低表达负担，而不是增加证明负担。
- “购物任务”关注购买前、购买中、购买后的自然路径，不应一开始要求货架、价格、促销审计。

## 本轮 prompt / rules 调整

### `research_rules.md`

新增：

- `21. 题目语气与开放性细则`
- `22. Case_001 受访者题面风格锚点`

重点约束：

- 优先使用“请和我们聊聊 / 请描画一下 / 回想一下最近一次 / 如果方便可以……”。
- 谨慎使用“列出 / 按优先级 / 至少 / 最多 / 每段用一行 / 必须 / 打分 1-5”。
- 具体性来自真实时刻、场景、人物、前后变化和原因，不来自硬塞窄标签。
- 括号示例要少用，每个板块只保留少量轻提示。

### `generation_logic.md`

在 Step 6 中补充：

- 详细题目要像受访者可自然回答的 Diary 任务，不要像研究员 checklist。
- 避免命令式和考试式措辞。
- 素材要求应作为降低文字负担的方式，不要每题强制上传。

### `case_cards/case_001_gum.md`

新增 `Respondent-Facing Style Anchors`：

- About Me 的题面锚点。
- Typical Day 的开放描画方式。
- Everyday Mouth-Related Item 的场景提示方式。
- 6-Day Diary 的自然记录方式。
- Shopping Task 的 vlog/journey 方式。

### `prompt.py`

系统 prompt 和生成 prompt 新增强约束：

- 不要把研究链路逐项翻译成 checklist。
- 避免“每段用一行 / 至少 / 最多 / 按优先级 / 打分 / 必须上传”。
- 不要每题都加括号解释。
- Case_001 类项目必须优先模仿正式 DG 的自然开放语气。

## 下一轮重点观察

重新跑 Case_001 后，重点看：

1. 板块 1 是否还过早进入口香糖/状态调节。
2. 板块 2 是否仍然像时间表或过度指定“精神下滑”。
3. 板块 3 是否还强制列数量、排序。
4. 板块 4 是否仍然过重、过度事件调查化。
5. 板块 5 是否从货架审计改为自然购买 journey。
6. 括号示例是否明显减少。
